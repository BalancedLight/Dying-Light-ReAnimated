# 0.6.0a1 — Dying Light 2 preview

This preview adds an explicit game profile and a bind-pose-corrected DL2 retarget path while preserving existing Dying Light 1 projects and defaults.

## Game profile selection

New projects can select:

```text
Dying Light 1
Dying Light 2
```

The choice controls the target bundle, root defaults, target validation, ANM2 format dispatch, and workspace guidance. Existing projects without a game-profile extension are inferred conservatively and remain loadable.

## DL2 target bundle

The bundled DL2 preview target contains:

```text
reference/dl2/player_shadow_caster.smd
reference/dl2/player_shadow_caster.crig
reference/dl2/0_m_fpp_farjump.anm2
```

`pelvis` is the primary pose/root bone. Independent IK and shadow-caster roots are preserved.

## Corrected FBX retargeting

The exact/subset-rig path now:

- permits source skeletons to contain extra facial, cloth, weapon, and helper bones;
- reads authoritative FBX bind-pose or skin `TransformLink` matrices;
- evaluates non-bone Model ancestors such as Blender's `Armature` wrapper;
- retains scene-axis and scale transforms carried by those ancestors;
- calculates a per-bone source-bind-to-target-bind basis correction;
- reconstructs target-local transforms from corrected target globals;
- keeps missing optional/helper targets at bind pose;
- rejects missing required deform bones with a readable report.

This replaces direct copying of FBX local rotations, which is unsafe when the FBX and target SMD describe the same joint positions with different bone-coordinate bases.

## FBX diagnostics

FBXs are checked on import and again before build. The report distinguishes errors from warnings and covers:

- unreadable or unsupported FBX data;
- missing skeleton or animation stacks;
- a selected stack with no moving skeleton curves;
- missing required target bones;
- source-skeleton supersets;
- hierarchy differences;
- multiple roots and helper-like roots;
- bind-pose and skin-cluster coverage;
- Armature/null wrappers carrying axis or scale conversion;
- suspicious unit scales or non-finite/singular matrices;
- Unicode-normalized name collisions;
- target-game and ANM2-template mismatches.

Blocking errors stop the build. Non-blocking findings are shown to the user and recorded in the build report.

## ANM2 format support

Dying Light 1 format 1 read/write remains unchanged.

The DL2 format-42 reader currently provides safe header and descriptor inspection. It recognizes the supplied stock sample as:

```text
format version:              42
frame count:                 6702
active descriptor count:     50
reference descriptor count:  139
```

Dynamic format-42 curve decoding and native format-42 writing are not enabled in this preview. The ANM2-to-FBX workspace reports that limitation instead of attempting to decode the file through the DL1 codec.

The FBX-to-ANM2 DL2 preview output currently uses the validated format-1 writer with the DL2 target descriptor profile. Treat it as a compatibility experiment until it is confirmed by DL2 DevTools/runtime testing.

## Compatibility

- Existing `.dlraproj` files remain loadable.
- The game selection is stored in the project's extension dictionary.
- Dying Light 1 remains the default for projects that cannot be identified safely.
- No existing body, mimic, model, RPack, or DL1 ANM2 code path is removed.
